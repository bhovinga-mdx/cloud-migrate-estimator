# Problem Statement

## Context

Mindex is a cloud services company offering application modernization, data analytics, cloud-native development, AI development, cloud migration, and cloud managed services. Sales engagements frequently require engineers to produce cost and timeline estimates for prospective clients.

## The Problem

High-cost engineering staff spend disproportionate time producing ballpark estimates for sales calls that have a low conversion rate. The current process:

1. A sales rep and 1-2 engineering reps join a call with the prospective client
2. The client presents a vague idea or RFP for a cloud engagement
3. After the call, engineers spend 1-5 days producing a ballpark estimate with cost and timeline
4. Only a fraction of these calls convert to actual engagements

The effort-to-ROI ratio on these estimates is poor. Senior engineers — the most expensive resource — are doing work that rarely leads to revenue.

## Who It Affects

- **Engineers:** Time pulled away from billable client work to produce speculative estimates
- **Sales reps:** Delayed turnaround on estimates reduces responsiveness to prospects
- **The business:** High cost of sales with low conversion yield

## Success Criteria

**Primary:** Reduce engineer time spent on initial sales estimates. The AI tool should produce an estimate that is "good enough" for internal review and client discussion without an engineer spending hours on it.

**Secondary:** Improve close rates by demonstrating AI capability to prospects — showing that Mindex leverages AI tools effectively.

## Current Process

```
Sales Call → Manual Notes → Engineer Review (1-5 days) → Ballpark Estimate
```

## Target Process (POC)

```
Sales Call → Transcript File → AI Pipeline (minutes) → Structured Estimate Report
```

## Long-Term Vision

- AI agent joins sales calls live and asks clarifying questions in real-time
- Support for all service types (app modernization, data analytics, cloud-native, AI dev, managed services)
- Historical data training to improve estimate accuracy over time
- External client research to inform estimates
- Output iteration — adjust estimates based on follow-up information
